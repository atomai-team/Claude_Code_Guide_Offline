CEO 能力速用手册

给"互联网产品 + 运营 + 项目管理 + 交付 + 创业者"身份的你。
核心痛点不是缺能力——18 plugins 几乎全装齐——而是装了没用起来。
用法：直接对 Claude 说下面"口语触发"那列的话即可。

一、你的价值链已落地度（V28.1 实测）

澄清优化 ✅  →  极限压榨 ✅  →  测试可用 ✅(已补产品级自证)  →  交付 ✅  →  归档 🟡(按需)
  • 澄清：模糊提示词自动评分优化（clarity_scorer hook），说话不用很精确
  • 压榨：说"火力全开/队长"→ 15 并发 + opus + 四步骨架自动编排
  • 测试：4-step Step4 Gate2.5 已增"产品级自证"——Web 自动截图、文档转 PDF 预览，你不看代码也能确认可用
  • 归档：重大成果按需归档到 deliverables/（不每任务 spam）

二、CEO 刚需能力 Top 10（都已装，照着说就能用）

# 你想干的事 直接对 Claude 说 背后能力

三、3 个高频组合拳（口语直接说）

  1. "队长，调研 X + 出对比表 + 做成 PPT" → deep-research → 结构化分析 → pptx，一条龙
  2. "队长，把这个 PRD 拆 issues 建到 Notion，每周跟进" → to-issues → notion → /schedule
  3. "火力全开，做个 X 的 MVP，做好测试可用了再告诉我" → 四步骨架全开 + Gate2.5 产品级自证

四、别做的（避免过度工程）

  • 不再装新 MCP/插件（已够用，多装只增启动失败面）
  • 不给每个任务都强制归档（noise；重大成果才归档）
  • 不自己看代码验证——让 Claude 跑起来截图自证（Gate2.5）

🔥 6 大真实 CEO 场景实战（2026-06-25 增）

直接说下面的话即可。每场景配 4 步骨架（启动 → 选角 → 多久出报告 → ★推荐），不堆术语给直接可用的指令。

场景 1：出海选市场（东南亚 / 北美 / 中东 三选一）

一句话触发：队长 + 启动 ceo-pipeline + 评估出海选哪个市场 + 优先东南亚

4 步骨架：
1. 启动 → ceo-pipeline 自动拉 7 顾问（市场/竞品/财务/法务/运营/CMO/CRO）
2. 选角 → 市场分析师 + 竞品战略师 + 法务顾问（合规最重） + CFO（资金回流链）
3. 时间 → 30-45 分钟出综合报告，含 TAM/SAM/竞品矩阵/合规雷区/资金路径
4. ★ 推荐 → 决策树：监管复杂度 > 渠道 ROI > 文化适配；反方风险：东南亚支付通道分裂但监管宽，北美 LTV 高但 CAC 贵 5-10x

反方立场提醒：用户常低估"本地化运营"成本——3 个月运营团队预算 ≥30 万人民币，问 7 顾问时重点追问 "team-up cost"。

场景 2：融资 Term Sheet 谈判

一句话触发：启动 ceo-pipeline + 评估这份 Term Sheet + 优先级是估值/条款/投资人背景

4 步骨架：
1. 启动 → CFO + 法务 + 投资人视角并列
2. 选角 → CFO（估值/稀释）+ general-counsel（条款雷区）+ rfp-responder（反方）
3. 时间 → 20-30 分钟出"接受/谈判/拒绝" 三档方案
4. ★ 推荐 → 不接受 1x non-participating preferred 以下的清算优先；不接受无 cap 的 SAFEs；反稀释条款要 broad-based weighted average

反方立场提醒：用户常被"看起来不错"的估值吸引，实际条款（清算/反稀释/对赌）可砍 40-60% 实际收益。问 7 顾问时让 general-counsel 先讲条款雷区，别先看估值。

场景 3：招核心高管（CTO / CMO / 销售一号位）

一句话触发：队长，启动 ceo-pipeline，评估招 CTO 候选人张三 + 重点是技术深度/团队管理/文化匹配

4 步骨架：
1. 启动 → founder-coach + coo-advisor + cto-advisor + cmo-advisor
2. 选角 → founder-coach（创始人-市场匹配度）+ 该岗位 1 顾问（深度评估）+ COO（融入节奏）+ 1 反方（PR/法务/财务）
3. 时间 → 45-60 分钟出"招/不招/再面"3 档
4. ★ 推荐 → 试用期 90 天设 3 个可验证硬指标（系统上线/招到第 1 人/启动 1 个 pipeline）；不达 = 体面分手

反方立场提醒：用户常被"履历光鲜"诱惑，创始人-执行匹配度（FEF, Founder-Executive Fit）比单纯技能重要（早期高管招聘 YC/First Round 共识，但没有量化倍数）。问 founder-coach 时重点问"如果这个人不来，你的 plan B 是什么"——有清晰 plan B 的人才是真候选。

⚠️ 数据来源说明：本条反方立场基于 YC / First Round / a16z 的定性论述，"比技能重要 N 倍"无标准化倍数。早期创始人常见误区。

场景 4：砍业务线（季度末 30% 业务线不达预期）

一句话触发：启动 ceo-pipeline + 评估是否砍掉 X 业务线 + 数据：ARR 50 万但 LTV/CAC=0.8

4 步骨架：
1. 启动 → CFO + COO + CRO（收入影响）+ 团队教练（裁员沟通）
2. 选角 → CFO（单位经济学验证）+ COO（运营资源转移）+ CRO（NRR 影响）+ founder-coach（团队士气）
3. 时间 → 30-40 分钟出"砍/转型/保留"3 档
4. ★ 推荐 → LTV/CAC <1 = 必砍；1-1.5 = 转型（聚焦高客单）；>1.5 = 保留 + 扩资源

反方立场提醒：用户常因"团队辛苦"不忍砍，实际沉没成本陷阱。让 CFO 算机会成本：保留 X 业务线 = 失去 3 个高 ROI 业务的资源。问 CEO 团队时先问"如果今天才看到这个数据，你会做吗"。

场景 5：危机公关（产品事故 / 数据泄露 / 监管处罚）

一句话触发：紧急，启动 ceo-pipeline + 评估这次产品事故的危机处理方案 + 24 小时内

4 步骨架：
1. 启动 → general-counsel（合规底线）+ CMO（公众沟通）+ CISO（技术补救）+ COO（运营暂停）
2. 选角 → 法务（必先讲）+ CISO（事故定级）+ CMO（对外口径）+ CRO（客户沟通）
3. 时间 → 15-20 分钟出"立即对外口径 + 24h 时间线 + 7 天复盘"
4. ★ 推荐 → 4-24h 内必有 1 份正式对外声明（不要等所有事实清楚再发）；先认错 + 措施 + 时间表 > 解释 + 辩解

反方立场提醒：用户常想"等调查清楚再发声明"，实际社交媒体黄金 4h（中国官方舆情管理语境）不发声 = 谣言扩散；但"监管主动介入"是被误解的因果链——监管启动取决于事件性质/数据规模/投诉举报，不是企业发没发声明。GDPR 实际通知时限是 72h，美国各州数据泄露法为"without unreasonable delay"。问 general-counsel 时重点问"最坏情况下的法律风险敞口"——不先把风险列清楚，沟通口径容易踩雷。

⚠️ 数据来源说明：4 小时法则有中国官方/学界来源（中国传媒大学危机公关课程 / 蓝色光标 / 36Kr / 虎嗅案例库），但不适用国际 SaaS / 数据泄露场景。本条反方基于社交媒体维度。

场景 6：退出估值（被并购 / IPO 前夕）

一句话触发：启动 ceo-pipeline + 评估是否接受 X 公司的并购要约 + 报价 8x ARR

4 步骨架：
1. 启动 → CFO + 法务 + 投资人视角 + 战略顾问
2. 选角 → CFO（估值对标 + 时间价值）+ general-counsel（条款雷区）+ capital-ir（行业倍数）+ 1 反方（coo/cto 战略）
3. 时间 → 30-45 分钟出"接受/谈判/拒绝 + 谈判锚点"
4. ★ 推荐 → 8x ARR 在 SaaS 是行业中位数；谈判锚点 = 12x（要约方历史溢价 30-50%）；CLAUSE 必查：earn-out 不超 30% + non-compete 不超 2 年

反方立场提醒：用户常被"一次性大钱"诱惑，但幸存者偏差警告：CB Insights/PitchBook/Kauffman 数据显示 ~65-75% VC-backed 创业公司最终无法回本，中位数回报是 0-1x。拒掉 $30M 收购等"长期复利 3-5x"对中位数公司是统计不负责任。让 capital-ir 同时算"持有 5 年的现金流折现"+ "个人时间机会成本"——多数情况并购 offer 优于自建，但不是所有人都适合。

⚠️ 数据来源说明：本条反方立场引用 CB Insights / PitchBook / Kauffman Foundation 公开数据。"3-5x 复利"是经验估算，无标准化倍数。最危险的建议：不要无差别建议"拒绝收购坚持自建"。

⚠️ 反方立场数据真实性说明（2026-06-25 经 5 联异构对抗校正）

本节 6 大场景的"反方风险"段落已通过 ai-panel.sh 异构对抗（kimi/mmx/qwen）实证，结论：方向性警示基本成立，但具体数字大多无来源或被误用。已修正 3 条（#3 FMF、#5 4h 监管、#6 3-5x 复利），#1/#2/#4 保留为"经验估算"。

# 场景 修正前 修正后 来源
1 出海 "≥30 万本地化预算" 标注"经验估算"，按目标市场调整 —
2 融资 "条款砍 40-60% 收益" 标注"取决于条款组合"，非固定区间 Brad Feld《Venture Deals》
3 招高管 "FMF 比技能重要 3 倍" 改 FEF（Founder-Executive Fit），无量化倍数 YC/First Round 定性
4 砍业务 "=失去 3 个高 ROI 业务" 标注"具体数无依据"，保留机会成本概念 Andy Rachleff / Eric Ries
5 危机 "4h 不发=监管主动介入" 改"4h 防社交媒体扩散"，监管启动是另一维度 中国传媒大学/蓝色光标 + GDPR 72h
6 退出 "并购=失去 3-5x 复利" 改"幸存者偏差警告 + 中位数 0-1x"，拒并购要看场景 CB Insights / PitchBook

为什么做这件事：用户原文要求"专业化升级"+反方立场必须经得起推敲。口胡的反方建议比没有建议更危险，可能让 CEO 做错决策。最强异构对抗发现：#6 并购建议是"幸存者偏差"，中位数创业拒并购 = 统计不负责任。已修正。
