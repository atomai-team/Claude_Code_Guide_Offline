一、协同关系全景 —— 你该用哪个？（最常问）

mycc 有 5 类"调度入口"，不重叠、各管一摊：

入口 口语触发 管什么 调用谁
队长 "队长/老大/活力全开/火力全开" 极限档总统筹（最高权限，可调下面全部） smart-flow-all → 任意 agent/team/skill
ceo-mode "评估/尽调/出海/商业可行性" 商业决策 5 商业顾问（市场/财务/法务/CMO/CTO…）
eng-experts "卡帕西/架构师/资深dev/严格审查" 工程深审 3 工程专家（见第二节）
start-work "开工/启动项目/做工程化" 工程化项目编排（规划→专家→验证→收口） planner + eng-experts + 3 验证
team-* "做app/安全审计/调研对比" 预制领域团队 team-fullstack/security/research…

关键关系（消除混乱）：
- 队长 ⊃ 全部：队长是总入口，内部会判断"这事该调商业顾问还是工程专家还是团队"，你说"队长"就不用记下面的。
- ceo(商业) ⊥ eng-experts(工程)：正交，一个管钱/市场/法务，一个管代码/架构。不会撞。
- start-work = 工程化项目的 facade：它内部调 eng-experts（阶段2专家）+ 验证团队，是"把多 agent 编排"这件事打包。单点深审用 eng-experts，完整项目用 start-work。
- team-* = 蓝图：预先编好的固定团队，适合"做个网站"这种标准活。

一句话决策：
- 不确定/想省心 → 说"队长"，它替你路由。
- 只想要某个专家看一眼 → 直接喊专家名（"卡帕西审查这段"）。
- 要把一个项目从头做到尾 → "开工"（start-work）。
- 纯商业评估 → ceo-mode。

二、工程化专家团队 —— 怎么启用（好用方案）

★ 推荐：口语直接喊，不用记英文名、不用记 skill 名。

你说 自动调到 他干什么
"卡帕西审查这段代码" / "严格看下质量" karpathy-discipline-expert KISS/YAGNI/DRY/命名/错误处理 5 维 + verify-before-assert
"资深dev 帮我决策" / "这架构选型对吗" senior-developer-mindset trade-off 表/SLA/SPOF/扩展性/成本 4 大决策框架
"架构师看下系统设计" / "出个 ADR" architect C4 模型/ADR/系统一致性（只读不改）
"三个专家一起看" eng-experts 三人并行 并行审 → 我综合去重 + ★推荐

工作机制：你说关键词 → eng-experts skill 自动激活 → 路由到 ~/.claude/agents/ 下对应 agent（Task 调度，opus）→ 输出含 file:line + ★推荐 + T0 口语档 + 工程化升级点（学习项）。

实例：
- "卡帕西看下我刚写的 hook" → 单点深审
- "开工做个 X，要专家把关" → start-work 阶段2 自动拉 3 专家并行

三、Manager 中心 —— 多 session 统一管理（2026-06-25 修通可用）

你开多个 session 处理 mycc 时，现在有 3 个能力自动帮你看住交叉：

能力 触发 你会看到
互可见 新 session 启动自动（SessionStart） 👥 活跃 session（近30min）: - <id> \| <项目> \| <在干什么>
主动查询 任意时刻跑 bash ~/.claude/bin/orchestrator-list.sh 同上，复盘"我现在几个 session 各在干嘛"
冲突提示 你说"改 X 文件"时自动（UserPromptSubmit） ⚠️[冲突] X 被 [其他session] 也在改
查重 你说新任务时自动 🔁[查重] 命中已有记忆条目（实义关键词才触发，不吵）

工作原理（修通后）：
- 每个 session 你发消息时，user-prompt-task-extractor.sh 把"本 session 在干什么"写到 ~/.omc/state/sessions/{id}/active.md。
- 你 Edit/Write 文件时，session-touched-recorder.sh 记录到 touched-files.json。
- 别的 session 启动/你提新任务时，读这些文件 → 互可见 + 冲突检测。
- TTL 30 分钟：只显示近 30min 活跃的，session 结束自动清（不会堆积 535 个历史）。

熔断：OMC_SKIP_HOOKS=1 可关全部。

四、智能路由 —— /goal、/ultrawork 等自动按需采用（需求 D）

你最早说"几乎没自动智能分配过"。机制是：

  • engineering-scenario-router skill 关键词触发 → 查 router-table.json（46 intents，SSoT） → 路由到 target_skill + execution_path（L0-L3/YOLO-L4 风险档）。
  • 透明路由：我会显式说"我要调 <skill> 因为你说了 <关键词>"，你能看到也能否决。
  • 未命中：我会说"我不确定该调哪个"，不静默瞎调。

怎么让它更主动：直接在话里带场景词（"深度调研X" / "全面升级" / "又报错" / "发布提PR"），命中即自动路由。完整词表见 router-table.json 或 docs/skill-index.md。

五、异构对抗融入策略（2026-06-25 · 触发策略而非到处硬接）

异构对抗 = 调外部 5 联 CLI（kimi/mmx/qwen/opencode/mavis，全非 Claude 模型）做对抗校验，破内部 Claude agent 同源盲思。入口 ai-panel "<问题>"（~/.local/bin）。

已接入（保持，~25 处）：claude-code-better(全面升级) / audit-all-better(审计) / smart-closure(高风险收口) / weekly-driver(周扫) / panel-harvest。

触发策略（2026-06-25 经外部 5 联对抗修订 · 三层，非单一"按需"）：

层 节点 机制
强制硬门控（不留判断层，防自我绕过） 不可逆变更 / 写 config·hook·cron·skill 注册表 / 架构写入 执行前硬拦截，无豁免按钮
条件触发（降判断门槛，用客观信号非"我觉得是架构决策"） 单次 >500 行 / 改 >3 模块 / 检测到"多 agent 零异议" 客观阈值触发
手动/不调 大战略决策（你本就警觉，外部缺上下文）/ 纯执行 / 对话 / ≤2文件可逆 手动按需 or 不调省 token

⚠️ qwen 关键洞察："触发分类器本身是另一个 Claude 调用 = 最偏的组件"。别让"判断是否架构决策"成为单点偏见——危险错误藏在"看起来 routine"的高频小调用里，不是大决策（大决策你本就警觉）。所以不可逆表面=硬接，不靠判断。

GAP 节点（外部一致收窄）：
- ✅ 唯一现在必补：omc-plan/ralplan consensus —— Planner→Architect→Critic 全 Claude 血统 = "一个训练分布戴三顶帽子/rigor theater"，同源盲思温室，外部杠杆最大。建议 Planner 输出后、Architect 前注入一个外部视角（流程门控非建议）。
- 🔸 ceo-pipeline —— 降级/可能过度：外部模型无领域上下文，对个人战略决策"廉价且自信地干扰"。仅月度/季度重大方向时手动跑，别自动。
- 🔸 self-evolve-loop —— 改机制：别"外部评估我的提案"（=validation theater，多半附和），改"让外部生成替代演化方案"再由你选（生成才对抗，评估不对抗）。

反模式（单人维护者 · 外部 4/4 点名，按危害排序）：
- ❌ #1 权威洗白/责任转移幻觉（最隐蔽）：你选的 5 个、prompt 的 5 个、挑的输出——"5 个外部模型同意"感觉更客观，实则一样偏。修复：停止问"这好吗"，强制外部只论证"这为什么错"（steelman 反方）；若不敢让外部论证反方，你要的是 validation 不是对抗。
- ❌ #2 决策瘫痪/取平均：5 个意见取中位数 = 严格更差。异构对抗价值在找矛盾不是平均。预先定义"分歧→做什么"，否则是拖延战术。
- ❌ #3 操作税：5 个外部 CLI 各有 failure mode/auth/version churn，单人维护。季度审计 value > maintenance tax，否则系统死于操作面膨胀。
- ❌ #4 监控/对抗工具自身不验证（2026-06-25 实证）：reality-check 工具误报 daemon"腐烂"，2 内部专家+2 外部 CLI 全被带偏 → 工具自己必先 verify-before-assert。
- 📐 反事实自测（qwen）：取 3 个月前一个你自信的决策，让外部模型不知你的结论地重构+论证反方。反方弱=对抗层有效；反方强=你蒙对了。

六、回滚与边界

  • Manager 中心全套：OMC_SKIP_HOOKS=1 关闭；回滚见各 hook 头部注释。
  • 安全不变量恒硬：配置防改 / 官方直连 / 改 hook 前 snapshot —— 任何档不放行。
  • 自动化健康自查：python3 ~/.claude/bin/automation-reality-check.py（虚假健康核查，launchctl 权威信号）。
