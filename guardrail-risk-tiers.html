Guardrail Risk-Tiers — 铁律风险分级（精简版 · 2026-06-24）

核心一句话：铁律分两物种——安全不变量（永硬）+ 流程 gate（按风险分级）。YOLO 调档只放流程 gate，不放安全不变量。

1. 两物种（不混）

Layer 0 安全不变量（YOLO 任何档不放行，违反=真实事故）：

  • 配置防改（settings.json model / enabledPlugins / env.ANTHROPIC_*）
  • 禁程序内 import 写函数 + 四件套
  • bash 禁 declare -A（本机 3.2.57 静默失效）
  • 官方直连（Anthropic 禁代理）
  • 改 hook / config 前先 snapshot

Layer 1 流程 gate（按风险分级，YOLO 可调档）：

  • 先 plan / 文件数 / 自动 commit / Writer≠Reviewer

2. 3 问定档（取代 5 维矩阵 · 实践足够）

判定风险档，答 3 问：

  1. 可逆？ — git/snapshot 可撤 = 可降档
  2. 碰 config/hook？ — 是 = 强制 L3
  3. ≤5 文件？ — 是 = 降档；否 = 升档
档 条件（3 问汇总） 动作
L0 1-2 文件 + 可撤 + 不碰 config/hook 直做 + 报告
L1 ≤5 文件 + 可撤 + 不碰 config/hook 验证 + clean 自动 commit（不 push）
L2 6-20 文件 / push / Notion 一句话简报即开
L3 不可逆 / 碰 settings·hook·cron / 架构 / security plan + 确认 + ai-panel

3. YOLO 调档（永不放行 Layer 0）

档 自动化
L1 保守 全 gate 生效
L2 标准 L0/L1 自动，L2 简报即开，L3 gate
L3 火力 L0-L2 自动，仅 L3 gate；clean 自动 commit
L4 极限 clean 自动 commit+push；仅不可逆/security gate

4. 收口决策（#6 智能收口 · 详见 skill smart-closure Step 2.5）

整体 ≤L1 + 验证过 + 意图对齐 → scoped commit 自动；≥L2 → 简报或确认。
