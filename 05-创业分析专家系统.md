# Claude Code 创业分析专家系统
## 多角色专家组 · 从想法到融资材料一站式分析
> 基于 OMC V52.2 · ceo-pipeline · 13 战略角色 · 2026-06-12

---

## 一、专家系统概述

### 1.1 系统定位

将 Claude Code 变成一个**7人专家小组**：每位专家从不同维度分析你的创业想法，相互质疑，最终给出综合建议。

```
你的想法/问题
     ↓
  [队长调度]
     ↓
┌───────────────────────────────────────┐
│  专家小组并行分析                      │
│  ① 市场分析师    → TAM/SAM/趋势      │
│  ② 竞争战略师    → 竞品/护城河       │
│  ③ 产品战略师    → PMF/MVP/路线图    │
│  ④ 财务建模师    → 单位经济学/融资   │
│  ⑤ 风险评估师    → SWOT/风险矩阵    │
│  ⑥ 增长黑客      → GTM/渠道/增长     │
│  ⑦ 投资人视角    → 可投性/杠杆点    │
└───────────────────────────────────────┘
     ↓
  综合裁决 + ★推荐
```

### 1.2 一句话激活

```
# 最简单的激活方式
"帮我用专家小组分析一下这个创业想法：[描述你的想法]"

# 完整激活（6阶段流水线）
"/ceo-pipeline 分析：[你的创业项目]"

# 指定专家角度
"用 CFO 视角分析这个商业模式的财务可行性"
"用投资人视角看这个项目，给出可投性评分"
```

## 二、专家角色详细说明

### 🔍 专家一：市场分析师（Market Analyst）

**职责**：市场规模、趋势判断、用户洞察

**激活方式**：
```
"帮我做一个市场分析：
 - 目标市场：[你的市场]
 - 需要知道：TAM/SAM/SOM、增长趋势、用户画像"
```

**产出物**：
```
市场分析报告（模板）：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📊 市场规模
  TAM（全球可寻址市场）：XXX亿
  SAM（可服务市场）：XXX亿  
  SOM（可获取市场）：XXX亿

📈 市场趋势（3-5年）
  驱动力：...
  阻力：...
  时机判断：现在进入合适吗？

👥 用户画像
  核心用户：[描述]
  痛点：[列举]
  现有解决方案的不满意点：[列举]

💡 市场机会
  未被满足的需求：...
  切入点建议：...
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**实际口语示例**：
```
"我做的是面向中小企业的 AI 客服 SaaS，
 目前市场上巨头有 Salesforce、Zendesk，
 帮我分析一下这个赛道的市场规模和机会点"
```

### ⚔️ 专家二：竞争战略师（Competitive Strategist）

**职责**：竞品矩阵、差异化定位、护城河建设

**激活方式**：
```
"帮我做竞品分析：
 主要竞品：[列举3-5个]
 我的产品：[描述]
 需要：功能对比/定价对比/差异化机会"
```

**产出物**：竞品对比矩阵 + 差异化定位图 + 护城河评估

**深度竞品调研**（用 firecrawl + deep-research）：
```
"帮我深度调研这5个竞品，每个都要：
 1. 核心功能截图描述
 2. 定价页分析
 3. 用户评价（App Store/G2/Trustpilot）
 4. 近期动态（融资/新功能）
 5. 我们可以差异化的点"
```

**竞品对比矩阵（示例格式）**：
```
                我们    竞品A   竞品B   竞品C
价格（月/席）    $49     $99     $29     $79
目标用户        中小企   企业    个人    中小企
核心功能A       ✅      ✅      ❌      ✅
核心功能B       ✅      ❌      ✅      ❌
AI 能力         ★★★★   ★★     ★★★    ★★★
集成数量        50+     200+    20      80
市场定位        AI优先   成熟    轻量    平衡

差异化优势：AI 优先 + 中小企价格 + 快速集成
```

### 🎯 专家三：产品战略师（Product Strategist）

**职责**：PMF 验证、MVP 定义、产品路线图

**激活方式**：
```
"帮我规划产品策略：
 想法：[描述]
 资源：[团队规模、时间、预算]
 目标：[什么叫成功]"
```

**PMF 验证框架**：
```
"帮我设计 PMF 验证实验：
 假设：用户愿意为 [功能] 付 [价格]
 验证方式：
 1. 用户访谈提纲（20 个问题）
 2. Sean Ellis PMF 调查（40% 会非常失望）
 3. 最小可验证实验设计
 4. PMF 达标标准"
```

**MVP 范围定义**：
```
"帮我定义 MVP：
 核心价值主张：[一句话]
 
 必须有（解决核心痛点）：
 - 功能1
 - 功能2
 
 不做（后续版本）：
 - 功能3
 - 功能4
 
 关键假设（MVP 要验证的）：
 - 假设1
 - 假设2
 
 成功标准（8 周内）：
 - 指标1 达到 X
 - 指标2 达到 Y"
```

### 💰 专家四：财务建模师（Financial Modeler）

**职责**：单位经济学、现金流预测、融资需求

**激活方式**：
```
"帮我做财务建模：
 商业模式：[SaaS/电商/服务等]
 目标市场：[规模]
 定价：[描述]
 当前状态：[是否有营收]"
```

**单位经济学分析**：
```
"分析我们的单位经济学：
 已知数据：
 - 客单价：XXX元/月
 - 获客渠道：[列举]
 - 大概获客成本：XX元
 
 帮我算：
 - CAC（客户获取成本）
 - LTV（客户终身价值）
 - LTV/CAC 比（目标 >3）
 - Payback Period（回收期）
 - 健康的目标值"
```

**18 个月财务预测（三情景）**：
```
"帮我做 18 个月财务模型：

悲观情景（市场推广遇阻）：
  Month 1-6: 月增长 5%，...
  Month 7-12: 月增长 8%，...
  
基准情景（按预期推进）：
  Month 1-6: 月增长 15%，...
  Month 7-12: 月增长 20%，...
  
乐观情景（超预期增长）：
  Month 1-6: 月增长 25%，...
  Month 7-12: 月增长 35%，...

给我：
- 每月 ARR/MRR 表
- 人员成本（按团队规模）
- 运营成本（云/营销/工具）
- 现金流和 Runway
- 何时需要融资"
```

**融资需求分析**：
```
"帮我分析融资需求：
 目标：做到 A 轮条件（ARR 100万美元）
 当前 Runway：X 个月
 
 给我：
 1. 需要融多少钱（18 个月 Runway）
 2. 用钱计划（人/产品/市场）
 3. 融资时间线建议
 4. 投前估值范围（基于类似公司对标）"
```

### ⚠️ 专家五：风险评估师（Risk Assessor）

**职责**：SWOT 分析、关键假设验证、风险矩阵

**激活方式**：
```
"帮我做风险评估，尽可能往坏处想，
 然后给出每个风险的应对预案"
```

**全面风险矩阵**：
```
帮我建立风险矩阵（概率×影响）：

关键假设验证：
□ 用户真的有这个痛点？（如何验证）
□ 用户愿意付钱吗？（如何验证）
□ 我们能低成本获客吗？（验证指标）
□ 团队能做到这件事吗？（能力评估）

风险分类：
1. 市场风险（市场不存在/太小/增长慢）
2. 竞争风险（巨头进场/快速跟进）
3. 技术风险（技术不可行/成本过高）
4. 执行风险（团队能力/文化/留存）
5. 资金风险（融资失败/现金流断裂）
6. 监管风险（政策变化/合规要求）

每个风险：
- 发生概率（1-5）
- 影响程度（1-5）
- 应对预案
- 监控指标
```

## 五、创业分析专家系统的工作原则

### 铁律对应

| 创业场景 | 对应铁律 | 具体执行 |
|---|---|---|
| 市场数据要真实 | 铁律 8（Verify-Before-Assert）| 市场规模引用官方/IDC/权威来源，禁用猜测 |
| 竞品数据要准确 | 铁律 8 + firecrawl | 竞品信息实时抓取，不靠印象 |
| 财务预测有依据 | 铁律 3（验证证据）| 每个假设都有来源数据支撑 |
| 风险不能美化 | 铁律 11（不可行硬约束）| 发现不可行必须明说，不能模糊拒绝 |
| 多视角对抗验证 | 铁律 10（Writer≠Reviewer）| 不同专家角色独立评估，禁自我确认 |
| 决策有推荐有理由 | output-contract.md | 每个分析必须有★推荐+理由+可执行动作 |

### 防"顾问综合征"规则

```
✅ 必须给具体数字（不只说"市场很大"）
✅ 必须有来源（研究报告/竞品数据/用户访谈）
✅ 必须有推荐（不只列选项，要给★推荐）
✅ 必须有下一步行动（不只给分析，要给可执行清单）
✅ 必须有验证方法（如何检验分析是否正确）

❌ 禁止："市场前景广阔，机遇与挑战并存"（空话）
❌ 禁止：只给选项不给推荐
❌ 禁止：分析完了没有具体行动项
❌ 禁止：数据没有来源
```
