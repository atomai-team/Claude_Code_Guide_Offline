一、MYCC 是什么

MYCC 是运行在 macOS 上的 Claude Code 智能多 Agent 编排层，面向商业创业者 + 边学边做的软件开发者：
- 核心：单调度器 + 四步骨架（Plan→Parallel→Force-Solve→Verify）+ 12 条铁律
- 能力：130 个技能 + 15 个工作流 + 47 个 Hook + 38 个 Agent + 12 铁律 + 12 行为规则
- 模型：Claude Opus 4.8 / Sonnet 4.6 / Haiku 4.5（Anthropic Max 官方直连）
- CLI：2.1.185 · OMC：4.14.7 · enabledPlugins=10(settings.json) + 35(settings.local.json 免疫层)

五、铁律（违反则本次报告作废）

  1. 禁编造：star/版本号/文件路径必实查（gh api 或页面读取），不凭记忆
  2. 只报增量：本机已有的跳过（对照上方"本机能力基线"）
  3. ≥2 源交叉：关键结论需多源验证；单源标"待验证"
  4. 禁通用建议：精简CLAUDE.md/用worktree/设deny这类本机已实施的——出现≥3条整报降级
  5. 输出=建议项：不写"请立即执行X"，改写"建议项：X（依据：Y）"
  6. 中文输出，技术术语保留英文
  7. 末尾附：可信度自评分（0-100）+ 未能独立核实清单

六、防错检查（提交前自检）

  • [ ] 版本哨兵在第一项？版本号实查？
  • [ ] 所有 star 数 GitHub API 实时查证？
  • [ ] 每条增量对照了"本机能力基线"？
  • [ ] 通用建议（本机已有的）全部剔除？
  • [ ] 无祈使指令，全部"建议项"？
