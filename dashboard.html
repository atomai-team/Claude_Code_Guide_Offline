5 分钟上手（零基础先看这个）

  1. 进项目：终端 cd 你的项目文件夹 → 输入 claude 回车
  2. 说人话：像聊天一样说要做什么，例 帮我把首页标题改成蓝色
  3. 看确认：它要改文件/跑命令时会问你 → y 同意 / n 拒绝 / 直接打字补充
  4. 遇报错：把报错整段贴给它，或说 又报错了，帮我查
  5. 换任务：先 /clear 重开；对话太长用 /compact 压缩

💡 口诀：说清楚要什么 + 让它给实跑证据 + 大改动先让它出计划

Agent / 队长 怎么用

子代理(subagent)=独立上下文的专属 Claude，调研/审查不污染你主对话。何时用：读大量文件、要独立审查者、并行提速。

直接说：用 subagent 调研 X / 派独立 agent 审查这段代码

队长模式：说 队长 / 火力全开 → 全权统筹，自动拆任务·派专家·并行。

自定义 agent：放 .claude/agents/名字.md(name/description/tools/model)。

工程化专家/协作入口怎么用 — 队长 / ceo / start-work / eng-experts

入口口语怎么说何时用
队长 说 队长，帮我做X 或 火力全开 日常多步骤任务，要它全权统筹四步工作流（最常用，首选）
ceo / 评估 说 评估这个项目 / 尽调 / 出海分析 商业/创业决策，要一组行业专家（市场/财务/法务等）出报告
start-work 说 开始做X项目 / start-work 启动一个真实软件/创业项目，要工程化协作中心带状态管理
eng-experts 说 找架构/安全/测试专家看看 要特定工程专家（架构师/安全/QA）做专项，非全流程

💡 不确定用哪个 → 直接说需求，队长会自动路由；详见 docs/工程化协作与多session-manager指南.md

# ── 基础操作 ──────────────────────────────────
claude --safe-mode              # 纯净启动（排障用，禁所有插件/hooks）
/cd ~/projects/xxx              # 切目录不重开 session
/btw 这个函数叫什么？             # 侧问，不污染主对话
/compact                        # 压缩长对话上下文

# ── 健康检查 ──────────────────────────────────
python3 ~/.claude/lib/mycc-stats.py          # 刷新全机数据 → 自动更新此页

/doctor                          # 诊断当前会话状态
omc update                       # 更新 OMC 插件

# ── 外部 AI 并发对抗 ──────────────────────────
bash ~/.claude/scripts/ai-panel.sh "调研 Next.js vs Nuxt" kimi mmx qwen opencode mavis
bash ~/.claude/scripts/ai-cli.sh kimi "分析行业报告"

/audit-all-better    # 30维诊断
/claude-code-better  # 诊断+修复
/harness-audit       # 六层评分
