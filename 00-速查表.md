### 🔗 融合使用 — 真实场景链路（4 类组合）

1. **"不知道找谁"** → 说 `super` → 主调度器列 1-3 候选 → 你选 → 路由到对应 skill
2. **"商业评估"** → 说 `CEO` → 7 顾问分析 → 想"工程化落地" → 说 `队长` → 全权执行
3. **"多 session 混乱"** → 经理自动显示活跃列表 → 想"收口" → 说 `秘书` 触发 weekly 巡检
4. **"完整 4 步"** → `秘书`（发现问题）→ `super`（路由到修）→ `队长`（极限修）→ `CEO`（评估收益）

### 🧠 super vs 队长 vs CEO vs 秘书 — 核心区别

- **super** = **路由层**（"我不知道该说什么" → 主调度器列候选让我选）
- **队长** = **执行层**（"我要全部权力" → 自动派 subagent 并行）
- **CEO** = **业务层**（"判断这生意值不值得" → 7 商业顾问）
- **秘书** = **自动化层**（"自动巡检 + 报告" → launchd 触发）

> 💡 **最佳实践**：先 `super` 摸清流程 → 想清楚用哪个 → 直接说名字（如"队长"）。

### 🚀 super 入口用法（2026-06-25 立 · 51 个意图 SSoT）

| `超级入口`（不带具体意图） | 主调度器显式问"你想做什么" → 你补一句 → 再路由 | 不静默瞎调，未命中就问 |
